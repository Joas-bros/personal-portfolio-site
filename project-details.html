Inventory Management System

A robust, full-stack application built to streamline warehouse operations and inventory tracking for small businesses.

The Problem and Solution

**Problem:** Our client was relying on error-prone spreadsheets to manage thousands of stock items, leading to frequent inventory discrepancies and lost sales. [cite: 44]

**Solution:** I designed and implemented a centralized, database-driven system that provides real-time stock updates, automated reorder alerts, and a secure way to track transactions. [cite: 43]

Key Features

  • **Real-time Stock Tracking:** Immediate updates on inventory levels across multiple locations. [cite: 45]
  • **User Role Management:** Granular permissions for warehouse staff, managers, and administrators. [cite: 45]
  • **Report Generation:** Automatic generation of PDF and CSV reports for sales, stock depletion, and supplier performance. [cite: 45]

Technologies Used

  • **Backend:** Python, Django Rest Framework [cite: 47]
  • **Database:** PostgreSQL [cite: 47]
  • **Frontend:** HTML5, CSS3, Vanilla JavaScript (for simple interactions) [cite: 47]
  • **Deployment:** Docker, AWS EC2 [cite: 47]

Key Learning Outcomes

This project was instrumental in mastering database optimization techniques. Specifically, I learned how to structure complex many-to-many relationships in PostgreSQL to handle high-volume transactions efficiently, which significantly improved report generation speed. [cite: 48]
